SmartSalim (개인용) - 제품 기획서
1. 문제 정의 (Problem Definition)
해결하려는 핵심 문제:

가정 내 발생하는 일상적인 문제 (욕실 곰팡이, 옷 얼룩, 쓰레기 배출 등) 를 인식할 때, 정보 검색, 앱 설치, 외부 상담을 거치는 번거로운 과정 때문에 해결이 미루어지는 문제.
**"지금 당장 내가 뭘 해야 할지"**를 몰라서 고민되고, 복잡한 매뉴얼을 읽어야 하는 인지 부하.
타겟 사용자:

초기 사용자는 "나 (개인사용자)": 집안일 관리에 지친 개인 사용자를 위해, 외부 전문가나 복잡한 앱이 필요 없는 즉각적이고 간결한 해결 도구를 원함.
비전 (Future): 개인 사용자의 생활 패턴 데이터를 기반으로, 누구나 사용할 수 있는 가정 관리 비서로 진화.
가치 제안:

"카메라 한 방으로 해결책이 생깁니다."
정보 습득이 아닌 행동 유도 (Action-oriented) 에 중점을 둠.
2. 사용자 시나리오 (User Scenario)
[시나리오 1: 욕실 곰팡이 발견] (Care)

상황: 샤워 후 욕실 거울이나 벽에 검은 곰팡이가 생겼을 때.
플로우:
앱 실행 후 카메라가 켜진 상태 (사진 전송 없음).
카메라로 욕실 벽면 (곰팡이 부위) 을 비춤.
앱이 실시간으로 "곰팡이"를 감지하여 카테고리 분류 (관리/Care).
즉시 결과: "곰팡이 제거제 A, B, C 중 가장 안전하고 효과적인 것은 이겁니다." + "지금 바로 닦아낼 수 있는 3 단계법."
사용자 행동: 제시된 3 단계법을 따라 즉시 닦음. (AS 상담이나 PDF 다운로드 없이 직접 해결)
[시나리오 2: 옷 기름 얼룩 처리] (Cleaning)

상황: 옷을 벗었을 때 옷에 기름 얼룩이 발견됨.
플로우:
카메라로 얼룩 부위 촬영 (실시간 뷰).
앱이 "유성 얼룩"으로 인식.
즉시 결과: "이 옷은 30 분 이내 세제 접촉이 필요합니다." + "가장 좋은 세제는 이 브랜드 (검색 결과 또는 링크 없이 구매처 안내) 입니다."
사용자 행동: 집에 있는 세제로 즉시 닦아냄.
[시나리오 3: 큰 쓰레기 버리기] (Disposal)

상황: 냉장고 서랍이나 오래된 전자제품을 버리고 싶을 때.
플로우:
카메라로 제품 바코드나 전체 모습을 비춤.
앱이 "폐기물"으로 분류.
즉시 결과: "이것은 2 일 뒤 수거됩니다. 지금 밖에서 기다리면 됩니다." 또는 "가장 가까운 수거장은 여기입니다."
사용자 행동: 지정된 시간/장소를 확인하고 배출 준비.
[시나리오 4: 유통기한 및 조립] (Knowledge)

상황: 마트에서 살거나 집안 구석진 곳에 둔 제품의 유통기한이 궁금할 때.
플로우:
카메라로 제품 라벨 (유통기한 또는 모델명) 비춤.
앱이 텍스트 인식하여 정보 추출.
즉시 결과: "유통기한: 2024.10.01 (과기한 주의)." 또는 "이 제품 조립 순서는: 1) 베이스 설치 -> 2) 모터 부착."
사용자 행동: 식량을 바로 섭취/처리하거나, 조립 시작.

4. 사용자 플로우 (User Flow) - 직관적 & 단단한 피드백
앱 실행 (Home Screen)

카메라 뷰프어 (실시간 뷰) 로 즉시 진입. (메뉴 화면 최소화)
"카메라가 뭐라고 생각하나요?"라는 문구로 사용자의 주의를 끔.
문제 제시 (Capture)

사용자가 카메라로 문제 상황 (곰팡이, 얼룩, 제품) 을 비침.
(무) 사진 전송: 데이터가 서버로 가는 과정 없이 즉시 처리됨.
분석 및 분류 (Analysis)

기기 내 AI 가 실시간으로 객체 감지 및 텍스트 인식.
카테고리 분류 (관리/청소/폐기/정보) 및 원인 추론 완료 (약 0.5~1 초 내).
행동 제안 (Immediate Action Suggestion)

결과 화면: 사진 업로드 전과는 달리, 바로 **"해결책 카드"**가 나타남.
구성:
문제 진단: "이건 곰팡이예요."
핵심 행동: "이제 이 세제로 30 초만 문지르면 됩니다."
실행 방법: 간단한 아이콘 3 단계로 그림.
AS/문서 배제: 불필요한 정보 (AS 번호, PDF 링크) 는 숨김. 오직 행동만 강조.
완료 및 피드백 (Completion)

사용자가 해결을 완료하면 (또는 다음 문제로 넘어가면) "완료" 또는 "다음 문제" 버튼으로 이동.
5. 제약사항 및 가정 (Constraints & Assumptions)
제약사항:

기기 성능 의존: 기기 내 AI 모델의 성능에 따라 인식 속도 및 정확도가 달라질 수 있음.
환경 노이즈: 극심한 빛의 변화나 매우 어두운 곳에서는 인식 오류 가능성.
데이터 프라이버시: 모든 분석이 기기 내에서 이루어지므로, 서버로 이미지 전송되지 않음.
가정:

사용자가 스마트폰 카메라에 익숙함.
사용자는 "해결책"을 즉시 실천할 의사가 있음.
텍스트 기반의 지각 (조립 매뉴얼 등) 은 기기 내 저장된 데이터베이스에서 즉시 추출됨.
6. 성공 기준 (Success Criteria)
인식 - 해결 시간: 카메라를 켰을 때부터 해결책을 얻는 데 3 초 이내 소요됨.
행동 수용율: 제시된 "지금 이렇게 해보세요"라는 제안에 사용자가 바로 **행동 (닦기, 버리기 등)**하는 비율이 90% 이상.
불필요 정보 제거: AS 번호나 문서 다운로드 버튼 클릭이 0 회 (사용자가 필요로 하지 않을 때).

8. 향후 기획 작업 및 계획 (Future Planning)
개인화 모델 최적화 (On-Device Learning):
개인 사용자의 생활 공간, 습관, 자주 사용하는 청소제 등을 학습하여 추천 정확도 및 행동 가이드를 더욱 세분화.
콘텐츠 라이브러리 확장:
조립 매뉴얼, 생활 꿀팁 등 텍스트/이미지 형태의 지식을 기기 내 데이터베이스로 지속적으로 추가.
다단계 문제 해결:
초기 단순 인식에서, 복잡한 문제 (예: 곰팡이가 심하면 "이제 심한데?" -> "전문가 상담 필요? 아니면 지금 이렇게 처리하세요?")로 확장.
가정 데이터 시각화 (선택 사항):
"지난달 청소 빈도", "폐기물 배출량" 등 개인 사용자를 위한 간단한 대시보드 제공 (사용자 의지에 따름).
